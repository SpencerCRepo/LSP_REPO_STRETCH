The OrderProcessor class has several issues that go against the object-oriented principles we covered. All the key fields like customerName, email, item, and price are public, so any code outside the class can change them directly. The processOrder needs to bec changed since it has so many tasks. If we need to change how tax works, update the file format, or add new discount rules, we have to edit this one long method and risk breaking other parts.  It’s the kind of design that becomes hard to maintain or extend as the system gets bigger
